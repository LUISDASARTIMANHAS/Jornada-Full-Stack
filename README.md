# Jornada Full Stack - Spotify Clone

## 1. Descrição do Projeto

Este projeto consiste no desenvolvimento de uma aplicação full-stack que replica as funcionalidades essenciais da plataforma de streaming de música Spotify. A iniciativa faz parte de uma jornada de aprendizado em programação full-stack, visando a aplicação prática de conceitos e tecnologias modernas de desenvolvimento web. A arquitetura do projeto é dividida em componentes de front-end e back-end, garantindo modularidade e escalabilidade.

## 2. Objetivo

O principal objetivo deste projeto é proporcionar uma experiência de aprendizado abrangente no desenvolvimento full-stack. Através da construção de um clone do Spotify, busca-se capacitar os desenvolvedores na integração de tecnologias de front-end e back-end, na gestão de bancos de dados e na implementação de interfaces de usuário dinâmicas e responsivas. O projeto visa solidificar conhecimentos em:

*   Desenvolvimento de interfaces de usuário interativas com React.
*   Criação de APIs RESTful robustas com Node.js e Express.js.
*   Gerenciamento de dados persistentes utilizando MongoDB.
*   Configuração de ambientes de desenvolvimento e build com Vite.
*   Implementação de práticas de código limpo e manutenção.

## 3. Proposta

A proposta metodológica deste projeto baseia-se em uma abordagem modular e iterativa. O desenvolvimento é segmentado em duas camadas principais:

*   **Front-end:** Responsável pela interface do usuário e pela interação direta com o cliente.
*   **Back-end:** Responsável pela lógica de negócios, persistência de dados e exposição de APIs para o front-end.

Essa separação permite o desenvolvimento e a manutenção independentes de cada componente, facilitando a colaboração e a escalabilidade. A integração entre as camadas é realizada através de requisições HTTP, seguindo os princípios de uma arquitetura de microserviços simplificada.

## 4. Tecnologias Utilizadas

### 4.1. Front-end

*   **React (v19.x):** Biblioteca JavaScript para construção de interfaces de usuário declarativas e baseadas em componentes.
*   **Vite (v7.x):** Ferramenta de build de próxima geração que oferece um ambiente de desenvolvimento extremamente rápido.
*   **React Router DOM (v7.x):** Biblioteca para roteamento declarativo em aplicações React, permitindo a navegação entre diferentes views.
*   **Axios (v1.x):** Cliente HTTP baseado em Promises para o navegador e Node.js, utilizado para fazer requisições à API back-end.
*   **Font Awesome (v7.x):** Conjunto de ferramentas e bibliotecas de ícones vetoriais escaláveis.
*   **ESLint:** Ferramenta de linting configurável para identificar e reportar padrões problemáticos encontrados no código JavaScript/JSX.

### 4.2. Back-end

*   **Node.js:** Ambiente de tempo de execução JavaScript assíncrono e orientado a eventos, construído sobre o motor V8 do Chrome.
*   **Express.js (v5.x):** Framework web rápido, flexível e minimalista para Node.js, utilizado para construir APIs RESTful.
*   **MongoDB (v7.x):** Banco de dados NoSQL orientado a documentos, escolhido pela sua flexibilidade e escalabilidade.
*   **npm-package-nodejs-utils-lda (v1.x):** Pacote de utilitários customizados para Node.js, desenvolvido para otimizar tarefas comuns no back-end.
